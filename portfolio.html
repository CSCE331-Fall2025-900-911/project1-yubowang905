Multithreaded Bank System

Developed a C++ program simulating a simple banking system where multiple users can simultaneously perform transactions. Implemented concurrency control using threads, mutexes, and condition variables to ensure data integrity while supporting parallel operations.

Challenges

  • Designing thread-safe access to shared resources (account balances).
  • Avoiding race conditions when multiple users attempted simultaneous transactions.
  • Handling potential deadlocks when different operations required overlapping resources.

Insights

  • Learned the importance of synchronization primitives in real-world applications.
  • Understood how improper concurrency design can lead to subtle bugs like data corruption or deadlocks.
  • Gained appreciation for testing under concurrent workloads to uncover issues not visible in single threaded scenarios.
